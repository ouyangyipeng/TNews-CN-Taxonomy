# 🎯 总体目标与超额交付（Beyond Full Marks）战略
本项目要求在 TNEWS 数据集上完成 15 分类的新闻主题预测。为了不仅“圆满完成”，还能达到“满分以上”的降维打击级别，你必须在以下几个维度做到极致：
1. **双架构+多分支对比**：不要只选其一，必须同时手动构建高质量的 `RNN + Attention`（如 BiLSTM + Bahdanau/Luong Attn）和 `Transformer`（如手动搭建 Encoder 堆叠）进行深度对比。
2. **多颗粒度分词消融**：不能仅停留在基础切分。实现基于字符级（Char）、词级（jieba）和子词级（BPE/WordPiece）的多种文本表示方式，并严格对比其在短文本上的表现。
3. **顶会级可视化（可解释性）**：除了 Accuracy 和 Macro-F1 折线图外，必须导出并绘制 **Attention Weights 热力图**（展示模型究竟关注了新闻标题的哪个词）和 **t-SNE 句子特征降维聚类图**。
4. **硬核错误分析**：结合混淆矩阵（Confusion Matrix），提取出容易混淆的具体新闻类别对（如“娱乐”与“影视”），并通过真实文本案例进行语言学和模型视角的深度归因剖析。
5. **极其规范的工程组织与 LaTeX 排版**：产出具有顶会排版质感的实验报告与模块化的工程代码。

# 🛠️ 框架与底层边界设定
- **允许使用的库**：允许使用 PyTorch 作为基础深度学习框架，以及 `jieba`、`transformers` (仅限使用其分词器 Tokenizer，绝对禁止直接调用现成的 `BertForSequenceClassification` 这种成品模型) 。
- **自研要求**：模型的主干网络（RNN的注意力机制计算、Transformer的Self-Attention与前馈层）必须通过 `torch.nn` 的基础算子自行拼装与搭建，在代码中体现你对网络结构的深度理解。
- **数据划分限制**：必须固定随机种子（Random Seed），严格从 `train.json` 划分为训练集和验证集（如 90% 训练，10% 验证）。由于官方 `test.json` 无标签，**必须将 `dev.json` 作为课程最终的独立测试集进行最终指标汇报**。

# 🔬 工作流流水线（必须严格遵循，循环演进）

## 阶段一：数据工程与探索性分析 (Data Engineering & EDA)
1. **计划生成**：在 `plans/` 下创建 `phase1_plan.md`。
2. **读取与清洗**：读取 UTF-8 编码的 JSON 数据。清理异常字符、统一全角半角符号。
3. **统计与洞察**：统计各类别样本分布的不平衡情况，统计短文本长度分布（给出 90/95/99 分位数以决定 Padding 的 `max_len`），将图表保存至 `report/figures/`。
4. **词表构建**：**严格遵守“只能使用训练集构建词表”的红线**。实现 `Dataset` 和 `DataLoader`，处理低频词与 OOV 问题。

## 阶段二：模型架构设计 (Architecture Engineering)
1. **计划生成**：在 `plans/` 下创建 `phase2_plan.md`。
2. **基线搭建**：在 `src/` 中模块化编写代码（如 `model.py`, `dataset.py`, `train.py`）。
3. **RNN+Attention 实现**：构建双向 LSTM/GRU。手动实现 Attention 机制（如 Context Vector 的计算权重公式须与数学定义一致），在代码注释中写明矩阵维度的变换逻辑。
4. **Transformer 实现**：实现基于多头自注意力（Multi-Head Self-Attention）和位置编码（Positional Encoding）的分类器。

## 阶段三：硬核训练与超参搜索 (Training & Tuning)
1. **计划生成**：在 `plans/` 下创建 `phase3_plan.md`。
2. **上机训练**：在 A100 上部署代码，执行多轮消融实验（至少涵盖不同学习率、不同切分方式、不同模型维度的组合）。
3. **反向侦察机制**：在验证集上严格监控 Loss，加入 Early Stopping 机制以应对过拟合。
4. **度量指标**：实现标准的评估逻辑，每次 Epoch 必须输出 Accuracy 和 Macro-F1（由于 TNEWS 存在一定长尾效应，Macro-F1 极度重要）。

## 阶段四：深入解析与可视化 (Analysis & Visualization)
1. **生成混淆矩阵**：绘制精美的 Seaborn 混淆矩阵热力图。
2. **模型可解释性实验**：提取一条验证集中的新闻短文本，如“*北京今日天气多云转晴*”，可视化出模型在判断其类别时，给哪些词汇分配了最大的 Attention Weight（导出热力图）。
3. **Bad Case 分析**：提取至少 3-5 个模型预测错误的典型样本，在 Markdown 中分析其错判逻辑（是因为文本太短、还是多义词引发的歧义？）。

## 阶段五：LaTeX 报告撰写 (Academic Report Writing)
1. 基于 `report/` 下的已有模板（如有），或自行编写极其专业的 `main.tex`。
2. 按照规定的八大结构撰写：任务描述、数据分析与预处理、模型设计说明、训练设置、实验结果、结果分析、心得体会、学术诚信说明。
3. **红线规则**：报告必须呈现“真实学术生”的口吻，严禁出现任何大模型套话（如“As an AI, 总体而言, 不可否认”）。心得体会部分应真诚探讨 CUDA OOM 调试、维度匹配难题或调参经验。包含必要的公式推导（如你选择的 Attention 计算公式）。

## 阶段六：清洗留痕与合规打包 (Cleanup & Packaging)
1. **反 AI 审查**：全局检索所有源码与注释，删除一切系统临时 Prompt 碎片、过于机械的注释风格。

3. 编写顶级开源项目级别的 `README.md`，写明“如何一键复现实验流程”。
4. 执行打包脚本，将包含源代码、README 和 PDF 的内容打包为 `2026ANN-NLP-学号-姓名.zip`。
